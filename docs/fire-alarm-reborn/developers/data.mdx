# Data Structures

This page documents the data structures in FAR.

***

## Alarm System

```lua
alarmSystem.id
alarmSystem.name
alarmSystem.status
alarmSystem.passcode
alarmSystem.location
alarmSystem.postalCode
alarmSystem.position.x
alarmSystem.position.y
alarmSystem.position.z
```

<Tabs>
	<TabItem value="alarmSystem.id" label="id" default>
		<p>`int`</p>
		<p>ID of the Alarm System.</p>
		<p>Example: `1`</p>
	</TabItem>

	<TabItem value="alarmSystem.name" label="name">
		<p>`string`</p>
		<p>Name of the Alarm System.</p>
		<p>Example: `Mission Row Police Station`</p>
	</TabItem>

	<TabItem value="alarmSystem.status" label="status">
		<p>`enum`</p>
		<p>Status of the Alarm System as an enum.</p>
		<p>Values:</p>
		- `0` - Invalid status
		- `1` - Entire system disabled
		- `2` - System ready / standby
		- `3` - Announcement in progress
		- `4` - WalkTest in progress
		- `5` - System has isolations
		- `6` - System in Alarm
		- `7` - System silenced
		- `8` - System test in progress

		<p>Example: `2`</p>
	</TabItem>

	<TabItem value="alarmSystem.passcode" label="passcode">
		<p>`string`</p>
		<p>Default passcode for Control Panels in this Alarm System.</p>
		<p>Example: `123`</p>
		:::warning
		Individual Control Panels can override this value, it may not correct for all Control Panels in the system.
		:::
	</TabItem>

	<TabItem value="alarmSystem.location" label="location">
		<p>`string`</p>
		<p>Human readable location of this Alarm System.</p>
		<p>Example: `Altee St, Mission Row`</p>
	</TabItem>

	<TabItem value="alarmSystem.postalCode" label="postalCode">
		<p>`string` | `null`</p>
		<p>Postal Code this Alarm System is located within.</p>
		<p>Example: `123`</p>
	</TabItem>

	<TabItem value="alarmSystem.postion" label="position">
		<p>`table`</p>
		<p>World coordinates of the center of this Alarm System.</p>
		<p>Example:</p>
		```lua
		local position = vector3(alarmSystem.position.x, alarmSystem.position.y, alarmSystem.position.z)
		```
	</TabItem>

</Tabs>
